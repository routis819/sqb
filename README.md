# SQB (SQL Builder)

A type-safe, modular SQL statement builder for Go, designed to decouple query
construction from execution.

## Features

- **Type-Safe Statements**: `Statement[T]` carries type information about the
  expected result row.
- **Structural Integrity**: The `StmtBuilder` enforces statement flow (e.g., must
  start with an `Opener` and end with a `Closer`).
- **Zero-Dependency Core**: The core `sqb` package has no external dependencies.
- **Pluggable Execution**: Database-specific execution logic is isolated in
  sub-packages (e.g., `run/sqlx`).

## Installation

```bash
go get github.com/routis819/sqb
```

## Architecture

The project is split into two main parts:

1.  **Core Builder (`sqb`)**: Handles the definition of clauses and the construction
    of the SQL string. It defines the `Statement[T]` type which encapsulates the
    generated query.
2.  **Runners (`run/...`)**: Provides integration with database libraries. Currently,
    `run/sqlx` provides support for [sqlx](https://github.com/jmoiron/sqlx).

## Usage

### 1. Define your Result Type

```go
type User struct {
    ID   int    `db:"id"`
    Name string `db:"name"`
}
```

### 2. Build a Statement

```go
import "github.com/routis819/sqb"

// Assuming you have types implementing Clause, Opener, and Closer:
builder := &sqb.StmtBuilder[User]{}
builder.MustAppend(SelectClause("id", "name"))
builder.MustAppend(FromClause("users"))
builder.MustAppend(WhereClause("id = :id"))
builder.MustAppend(LimitClause(1))

stmt := builder.MustStmt()
fmt.Println(stmt.String()) // "SELECT id, name FROM users WHERE id = :id LIMIT 1;"
```

### 3. Execute with sqlx

```go
import (
    "github.com/jmoiron/sqlx"
    runsqlx "github.com/routis819/sqb/run/sqlx"
)

func GetUser(db *sqlx.DB, stmt sqb.Statement[User], id int) (*User, error) {
    args := map[string]any{"id": id}

    // Uses the dedicated sqlx runner
    results, err := runsqlx.NamedQueryStruct(db, stmt, args)
    if err != nil {
        return nil, err
    }

    if len(results) == 0 {
        return nil, nil
    }
    return results[0], nil
}
```

## Advanced: Implementing Clauses

To extend the builder, implement the `Clause` interface:

```go
type Clause interface {
    AcceptNext(Clause) bool // Validation logic for what can follow this clause
    String() string         // The SQL fragment generated by this clause
}
```

Implement `Opener` for clauses that can start a statement (like `SELECT`) and
`Closer` for clauses that can terminate a statement (like `LIMIT` or `ORDER BY`).
